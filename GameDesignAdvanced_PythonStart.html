    GAME #1: A simple Guessing Game
    A simple way to approach a (limited) text adventure is to repeatedly nest if-statements. This is relatively easy to code, but non-intuitive to read/edit afetrwards.

    Consider this script: GuessingGame.txt
    (see pythonWeek1.zip, or copy txt into new file and save as .py)

    NOTE:
  • In the WIN condition, the + + around the "name" variable is string concatenation.
  • In the LOSE condition, the comma after the string allows a different type (int) to be displayed there.

    EXERCISE: How can we make code that is easier to understand and revise?

    • The full script:
    # Description: A number guessing game by Mr McNally

    import random

    name = input("Hello, what is your name? ")
    answerNum = random.randint(1, 100)
    print("Hi " + name + ", I'm thinking of a number 1-100.")
    guessesTaken = 0
    while guessesTaken < 5:
        guess = input("Enter a guess: ")
        guess = int(guess)
        guessesTaken = guessesTaken + 1
        if guess < answerNum:
            print("That was too low.")
        elif guess > answerNum:
            print("That was too high!")
        else:
            break

    # once the while loop is broken
    if guess == answerNum:
        print("You win! Congrats, " + name + ", you guessed right")
    else:
            print("You lose. The right answer was ", answerNum)
