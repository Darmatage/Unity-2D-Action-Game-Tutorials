UNITY TUTORIAL: Canvas UI Tools
    Unity Canvas is a 2D space to hold HUD elements ("Heads Up Display"): Text, Images, Buttons, Sliders, and more.
    In the Scene view the Canvas appears over-large, but in the Game View it maps over the visible screen and tracks with camera movement.
    Text objects can be static (display message and not change) or can be updated by a script.
    Buttons and Sliders include Event Triggers to call (activate) a script on another object.
    STANDARD CANVAS SET-UP: In Inspector set Canvas Scaler > UI Scale Mode from "Constant Pixel Size" to "Scale With Screen Size" and Reference Resolution = 1280x720

 
[A]: BASIC TEXT UPDATE:
1. Create an Empty Game Object: GameObject > CreateEmpty, name GameControlObj.
In the Inspector apply the tag GameController.

Create a new C# script named "GameController". Add script to GameControlObj.
Doubleclick script to open in editor. Be sure to add the using UnityEngine.UI namespace to the top:

GameController.cs:
using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UI;

public class GameController :MonoBehaviour {
       public GameObject textGameObject;
       private int score;

       void Start () {
            score = 0;
            UpdateScore ();
             }

       public void AddScore (int newScoreValue) {
             score += newScoreValue;
             UpdateScore ();
             }

       void UpdateScore () {
             Text scoreTextB = textGameObject.GetComponent<Text>();
              scoreTextB.text = "Score: " + score;
             }
         }

2. Create a Canvas Text object: GameObject > UI >Legacy > Text
(NOTE: Canvas appears normal in Game view and crazy large in Scene view).
     a. In Inspector, change text field to "Score?" and set Pos X and Pos Y to a corner.
Set Width and Height larger to increase text field (double or more)
     b. Select the GameControlObj and drag the Text object into the empty public Text script slot (in the GameController script component).

3. Finally, any object in your Scene can access this AddScore method to update the score:
#1. Add a private variable to hold the GameController object.
#2. Add a Start method to find the actual GameController object and assign it to the private variable (Assign first to a temporary GameObjectTemp variable). This only works if the GameController component already exists, assigned to the GameController object (step 1).
#3. Add a line (like to an OnTriggerEnter function) to call AddScore inside of GameController script:

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class HitObjectToAddScore : MonoBehaviour {

       private GameController gameController;    // #1 variable to hold GameController

       void Start () {                                              // #2 assign actual Game Controller to var
             GameObject gameCtrlTemp = GameObject.FindWithTag ("GameController");
             if (gameCtrlTemp != null) {
                 gameController = gameCtrlTemp.GetComponent <GameController>();
                  }
             if (gameCtrlTemp == null) {
                  Debug.Log ("Cannot find 'GameController' script");
                  }
             }

       void OnTriggerEnter(Collider other){
             if (other.gameObject.tag == "Player") {
                   gameController.AddScore (1);             // #3 update score through AddScore()
                  }
             }
}



 
[B]: CANVAS BUTTONS: (video)
1. Add a Game Object > UI > Canvas to create a transparent space that will hold our UI elements.

2. In the Hierarchy select the Canvas, RightClick and add a UI > Button.
Position as desired. Note the Button also includes a Text child Component, which can be changed by selecting and typing into the field in the Inspector. Type "My Text".
With the Button selected, change the Pressed Color.
Hit Play to see the result when the button is clicked. Turn off Play.

3. Create a script and add it to the Button, ButtonHandler.cs:
using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using System.UI;

public class ButtonHandler : MonoBehaviour {

     public void SetText(string text){
            Text txt = transform.Find("Text").GetComponent<Text>();
            txt.text = text;
      }
}

4. Select the Button and add an OnClick event: hit the [+] button in the On Click () box.
Click [None (Obj)] to add the Button, and click [No Function] to choose Button Handler > SetText().
In the text box that appears below function, type "New Text". Hit Play to see the text change when clicked.
5. USE A BUTTON TO SWITCH SCENES (like to level #1):
    a. Use the Script below (note SceneManagement namespace).
    b. change OnClick() function to ButtonHandler > PlayGame().
using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.SceneManagement;

public class ButtonHandler : MonoBehaviour {

      public void PlayGame () {
            SceneManager.LoadScene("Level1");  //uses level name
      }
}


 
[C]: VOLUME CONTROL SLIDER (source video, sample audio)
This tutorial offers more accurate logarithmic control over volume.

1. Create a Window > Audio > Audio Mixer (or hit [Ctrl/Cmd]+[8]) fro your project (is created in a new panel, exists for all scenes).
      a. Hit the [+] to create a new Mixer, name it MyMusicMixer.
      b. Expose volume as a variable:
          Select Mixer Groups > Master.
          In Inspector RightClick Volume to choose:
              Expose 'Volume (of Master)' to Script.
          In upper right corner of Mixer click Exposed Parameters
          rolldown, change name "Volume of Master" to "MusicVolume".

2. Create an Empty Game Object to hold the game audio.
      a. Drag audio clip .WAV onto it for an Audio Source Component.
      b. Under AudioSource > Output hit hotspot to add MyMusicMixer.

3. Create a GameObject > UI > Slider.
      a. Set Min Value to 0.0001 (leave Max Value at 1).
      b. Set Value = 1.
      c. Add this C# script, called "SetVolume.cs":
using UnityEngine;
using System.Collections;
using UnityEngine.Audio;

public class SetVolume : MonoBehaviour{

 public AudioMixer mix;

 public void SetLevel (float slideVal){
    mix.SetFloat("MusicVolume", Mathf.Log10 (slideVal) * 20);
 }
}

When the saved script loads in Unity on the Slider:
a. At the "mixer" script slot hit the HotSpot to add MyMusicMixer.
b. Above, in OnValueChanged event trigger, hit [+] to add an event.
c. Drag SetVolume Component up into "None" slot (adds Slider to itself).
d. Click [No Function], set SetVolume > Dynamic float SetLevel (use the top one, NOT the bottom one!).

Hit Play to scrub slider and change volume!


 
[D]: PAUSE MENU:
1. Create the canvas:
a. Create a Canvas object: GameObject > UI > Canvas. Leave it named "Canvas". In Inspector set to Scale with Screen Size and choose 1280x720.
b. With Canvas selected create an UI > Image object, name it "PauseMenu".Set size to 1280x720, centered (x and y = 0).

d. Apply 6 GameObject >UI elements below PauseMenu:

        Duplicate ([Ctrl/Cmd] + [D]) Button (#2): set text to "RESTART".
        Duplicate to make Button (#3): set text to "QUIT"
        GameObject > UI > Legacy > Text set text field to "PAUSE MENU", locate top.
                 Size object and font large.
        GameObject > UI > Legacy >Text set text field to "VOLUME:", locate middle.
                 Size object and font large.
        GameObject > UI > Slider. Add a new Tag to the slider: "PauseMenuSlider".
                Set Slider Min Value to 0.0001 (leave Max Value at 1) and Value = 1.
e. Select the Buttons, and in the Inspector, under Image, click Color and set Alpha to zero (if you did not add a custom image). In the text object below each button set font Size = 25 and color to white.

2. Create the Audio Objects (as explained in the Volume Control tutorial):
a. Open Window > Audio > AudioMixer, hit [+] to create new mixer and name it "MyMusicMixer".
b. Expose volume as a variable: Select Mixer Groups > Master. In Inspector RightClick Volume to choose: "Expose 'Volume (of Master)' to Script." In upper right corner of Mixer click Exposed Parameters rolldown, change name "Volume of Master" to "MusicVolume".
c. Create an Empty G.O., name it "AudioManager." Drag an audio file onto this object to add it in an Audio Source Component (music or other ambient sounds, as .MP3 or .WAV). Under the AudioSource Component > Output hit the hotspot to add MyMusicMixer. Drag the AudioManager object into the Project panel to make it a Prefab (ideally in a folder named Prefabs) so you can easily add it to other Scenes, where the music file can be changed.

3. Add script "PauseMenu.cs" to the parent Canvas.
a. Into the "pauseMenuUI" script slot drag the PauseMenu Canvas object.
b. For the "mixer" script slot hit the hotspot to load your Audio Mixer object.

PauseMenu.cs:
using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UI;
using UnityEngine.SceneManagement;
using UnityEngine.Audio;

public class PauseMenu : MonoBehaviour  {

        public static bool GameisPaused = false;
        public GameObject pauseMenuUI;
        public AudioMixer mixer;
        public static float volumeLevel = 1.0f;
        private Slider sliderVolumeCtrl;

        void Awake (){
                SetLevel (volumeLevel);
                GameObject sliderTemp = GameObject.FindWithTag("PauseMenuSlider");
                if (sliderTemp != null){
                        sliderVolumeCtrl = sliderTemp.GetComponent<Slider>();
                        sliderVolumeCtrl.value = volumeLevel;
                }
        }

        void Start (){
                pauseMenuUI.SetActive(false);
        }

        void Update (){
                if (Input.GetKeyDown(KeyCode.Escape)){
                        if (GameisPaused){
                                Resume();
                        }
                        else{
                                Pause();
                        }
                }
        }

        void Pause(){
                pauseMenuUI.SetActive(true);
                Time.timeScale = 0f;
                GameisPaused = true;
        }

        public void Resume(){
                pauseMenuUI.SetActive(false);
                Time.timeScale = 1f;
                GameisPaused = false;
        }

        public void Restart(){
                Time.timeScale = 1f;
                //restart the game:
                //SceneManager.LoadScene (SceneManager.GetActiveScene().buildIndex);
SceneManager.LoadScene ("MainMenu");
        }

      public void QuitGame() {
                #if UNITY_EDITOR
                UnityEditor.EditorApplication.isPlaying = false;
                #else
                Application.Quit();
                #endif
      }

        public void SetLevel (float sliderValue){
                mixer.SetFloat("MusicVolume", Mathf.Log10 (sliderValue) * 20);
                volumeLevel = sliderValue;
        }
}

3. Set up functionality for each UI element:
a. Select each Button, in Inspector find the OnClick field, hit [+].
b. Drag the parent Canvas object into the "None" slot.
c. Click [No Function] to find the PauseMenu() script and choose the corresponding function:
        QuitGame() for Quit
        Restart() for Restart
        Resume() for Resume
d. Select the Slider. In Inspector find the OnValueChanged field and hit [+]. Drag Canvas into None, click [No Function] to find the PauseMenu() script and at the top of the options under Dynamic float choose "SetLevel"
(if you have not already done so, follow the Volume Control tutorial using the PauseMenu slider: expose the Master Volume variable, name it MusicVolume, set the Audio Source Output to mixer, etc).

To playtest, hit Play and at any time tap the [Esc] key.

FINALLY, REPLICATE: This script maintains persistant audio levels for all scenes that include the Pause menu. Drag it into the Project panel to make a Prefab, and then drag into all game and menu scenes under their Canvas, add the script tom the Canvas, and re-connect the buttons and slider.

IMPORTANT: If you already have an Update() function in the GameHandler script using the [ESC] key to quite the game, you need to delete or comment out this code, or otherwise every time you try to hit [ESC] to pause the game in the actual build, it will instead close the game.

    [STEP 2]: Create script for HealthBar:
    Add a script to take damage. Note that health can be 0-100 or 0-200, etc, but fillAmount only sees 0-1, so we need to adjust for that: divide current health by the starting full health value before it is displayed:

    RightClick in the Project panel to create new C# script "HealthBarThirdPerson.cs."
    Drag the script onto the Canvas object for your healthbar, then drag HealthBar into the image script slot.

    HealthBarThirdPerson.cs
    using System.Collections;
    using System.Collections.Generic;
    using UnityEngine;
    using UnityEngine.UI;
    using UnityEngine.SceneManagement;

    public class HealthBarThirdPerson : MonoBehaviour {

          public float startHealth = 100;
          private float health;
          //public GameObject deathEffect;
          public Image healthBar;
          public Color healthyColor = new Color(0.3f, 0.8f, 0.3f);
          public Color unhealthyColor = new Color(0.8f, 0.3f, 0.3f);

                //temporary time variables:
          public float timeToDamage = 5f;
          private float theTimer;
          public float damageAmt = 10f;

          private void Start () {
                health = startHealth;
                theTimer= timeToDamage;
          }

    // this timer is just to test damage. Comment-out when no longer needed
          void FixedUpdate () {
                theTimer -= Time.deltaTime;
                if (theTimer <= 0) {
                      TakeDamage(damageAmt);
                      theTimer = timeToDamage;
                }
          }

          public void SetColor(Color newColor){
                healthBar.GetComponent<Image>().color = newColor;
          }

          public void TakeDamage (float amount){
                health -= amount;
                healthBar.fillAmount = health / startHealth;
                //turn red at low health:
                if (health < 0.3f){
                      if ((health * 100f) % 3 <= 0){
                            SetColor(Color.white);
                            Die();
                      }
                      else {
                            SetColor(unhealthyColor);
                      }
                }
                else {
                      SetColor(healthyColor);
                }
          }



          public void Die(){

    using System.Collections;
    using System.Collections.Generic;
    using UnityEngine;

    public class GamerHandler : MonoBehaviour {

                      else {
                            healthBar.SetColor(Color.red);
                      }
                }
                else {
                      healthBar.SetColor(Color.green);
                }
          }
    }

    [F]: BASIC INVENTORY SYSTEM #1: A very simple, hard-coded system for an Inventory.

    This tutorial uses PNG Sprites for:
    5 unique pickup items, 1 container, 1 player.
    Click here to download a zip of all 7 images.
    SETUP: 2D Player Move and Tilemapping
    		   
          		 

    1. CANVAS: If you do not already have a GameObject > UI > Canvas object in your Hierarchy, create one.
    a. In Inspector, set Canvas Scaler > UI Scale Mode from "Constant Pixel Size" to "Scale With Screen Size" and Reference Resolution to 1280 x 720.
    b. Create a GameObject > Empty GO, name it "InventoryMenu". Parent it under the main Canvas.
    c. Add a GameObject > UI > Image. Name it "InventoryBG" and use the Rect tool to shape it into a short, long rectangle near canvas bottom. In Inspector set opacity to medium and choose a color.
    d. Add another UI > Image and use Rect tool to size and position as an Item in the Inventory (NOTE: visibility is reverse of Hierarchy order, so lowest item in Hierarchy is shown in scene on top).
    e. Include a GameObject > UI > Legacy > Text object to track the number of coins found.

    2. Create a PREFAB of the intended container: Drag the container sprite into the Hierarchy, name it ItemContainerPrefab.     3. GAME INVENTORY: If you do not already have a GameHandler object: create a GameObject > Empty GO, name it "GameHandler". In Inspector set the tag "GameHandler" and add the following script (I prefer to keep this script separate from the GameHandler script that manages other player stats, for clarity).

    NOTE: This hard-coded Inventory system requires two variables for each item (a more elegant solution could store all items in an array):
  • A bool variable for each inventory item, to set the state (whether they are collected or not: on or off)
  • A GameObject variable for each inventory image, to make them invisible/visible.

    In the Start() function note the InventoryMenu is set visible and all inventory items are set not visible. We can also run the InventoryDisplay() function to set the Inventory Image Items to not visible.

    In addition to the InventoryDisplay() function, we have an InventoryAdd() function to call by the ItemContainer.cs script, an     InventoryRemove() function to call if an item is "spent", and a CoinChange() function to add coins when found or subtract coins for a purchase (NOTE: any action to use an item or spend coins should first check if their bool is true or if the amount is sufficient for the purchase).

    GameInventory.cs
    using System.Collections.Generic;
    using System.Collections;
    using UnityEngine;
    using UnityEngine.UI;

    public class GameInventory : MonoBehaviour {

          //5 Inventory Items:
          public GameObject InventoryMenu;

          public static bool item1bool = false;
          public static bool item2bool = false;
          public static bool item3bool = false;
          public static bool item4bool = false;
          public static bool item5bool = false;
          public static int coins = 0;

          public GameObject item1image;
          public GameObject item2image;
          public GameObject item3image;
          public GameObject item4image;
          public GameObject item5image;
          public GameObject coinText;

          void Start(){
                InventoryMenu.SetActive(true);
                item1image.SetActive(false);
                item2image.SetActive(false);
                item3image.SetActive(false);
                item4image.SetActive(false);
                item5image.SetActive(false);
                InventoryDisplay();
          }

          void InventoryDisplay(){
                if (item1bool == true) {item1image.SetActive(true);} else {item1image.SetActive(false);}
                if (item2bool == true) {item2image.SetActive(true);} else {item2image.SetActive(false);}
                if (item3bool == true) {item3image.SetActive(true);} else {item3image.SetActive(false);}
                if (item4bool == true) {item4image.SetActive(true);} else {item4image.SetActive(false);}
                if (item5bool == true) {item5image.SetActive(true);} else {item5image.SetActive(false);}

                Text coinTextB = coinText.GetComponent ();
                coinTextB.text = ("COINS: " + coins);
          }

          public void InventoryAdd(string item){
                string foundItemName = item;
                if (foundItemName == "item1") {item1bool = true;}
                else if (foundItemName == "item2") {item2bool = true;}
                else if (foundItemName == "item3") {item3bool = true;}
                else if (foundItemName == "item4") {item4bool = true;}
                else if (foundItemName == "item5") {item5bool = true;}
                InventoryDisplay();
          }

          public void InventoryRemove(string item){
                string itemRemove = item;
                if (itemRemove == "item1") {item1bool =false;}
                else if (itemRemove == "item2") {item2bool =false;}
                else if (itemRemove == "item3") {item3bool =false;}
                else if (itemRemove == "item4") {item4bool =false;}
                else if (itemRemove == "item5") {item5bool =false;}
                InventoryDisplay();
          }

          public void CoinChange(int amount){
                coins +=amount;
                InventoryDisplay();
          }
    }

    [G]: WARNING VIGNETTE / DISTANCE CLACULATOR:
    Warn your player of impending doom with this script! Uses the distance from player to enemy to determine whether to start the warning, and get more intense the closer the enemy is within that range.

    Create a Canvas object (if on does not already exist).
    In Inspector, set Canvas Scaler > UI Scale Mode from "Constant Pixel Size" to "Scale With Screen Size" and Reference Resolution to 1280x720.

    Apply this script to a Canvas Image object, with vignette (framing) art applied.

    EnemyWarning.cs:
    using System.Collections;
    using System.Collections.Generic;
    using UnityEngine;
    using UnityEngine.UI;

    public class     EnemyWarning : MonoBehaviour  {

            public GameObject player;
            public GameObject enemy;
            public float distance;
            public float enemyRange = 20f;

            void Update() {
                    distance = Vector3.Distance(player.transform.position, enemy.transform.position);
                    if (distance <= 20f) {
                            float alphaChange = (1 - (distance / 20) + 0.1f);
                            Debug.Log("AlphaChange = " + alphaChange);
                            GetComponent <Image>().color = new Color(1, 1, 1, alphaChange);
                    }
                    else {
                            GetComponent <Image>().color = new Color(1, 1, 1, 0);
                    }
            }
    }

    using System.Collections;
    using System.Collections.Generic;
    using UnityEngine;

    public class     ChangeAlphaKeys : MonoBehaviour {

            public KeyCode decreaseAlpha;

            void Update(){
                    if (Input.GetKeyDown (increaseAlpha))
                            alphaLevel += 0.05f;

                    if (Input.GetKeyDown (decreaseAlpha))
                            alphaLevel -= 0.05f;

                    GetComponent<    SpriteRenderer>().color = new Color(1,1,1, alphaLevel);
            }
    }

    SPEED: Need to show the speed something travels, like a vehicle or character?
    Calculate Speed with this script, and display movingSpeed as needed (source)

    CalcSpeed.cs:
    using System.Collections;
    using System.Collections.Generic;
    using UnityEngine;

    public class     CalcSpeed : MonoBehaviour  {

            void Start(){
                    StartCoroutine(CalcVelocity());
            }

            IEnumerator CalcVelocity(){
                    while(Application.isPlaying){
                            Vector3 lastPos = this.transform.position;
                            yield return new WaitForFixedUpdate();
                            movingSpeed = Mathf.RoundToInt(Vector3.Distance(this.transform.position,lastPos)/Time.fixedDeltaTime);
                            Debug.Log("Speed of char is: " + movingSpeed);
                    }
            }
    }
