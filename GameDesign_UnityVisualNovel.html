PART 1: PROJECT EXPLANATION


How to Design a Visual Novel Short Story

PLANNING YOUR STORY:
A Branching Narrative Game is a chance to flex storytelling muscles and invite players to explore a world of choices. The design of this system turns some choices into places: big decisions are about going somewhere else for a new interaction. Smaller "story" choices affect the flow of the conversation in a Scene, and help the player define their character.

Consider this OUTLINE and FLOW CHART for a short interactive story about a turtle who has just hatched from its shell. The first choice is to decide what to do next. Does it follow a girl turtle DOWN the beach to the one big light, or the boy turtle UP the beach to the many lights, or its own nose to the sweet smell ALONG the beach?

Each of these Scene Change choices leads to places (new Unity Scenes) with choices that can lead to death (and being eaten by vultures), or chances to revisit old choices, and one slim path for victory.

The text of every Scene is meant to be a dialogue, so in addition to the player there should always be at least one other character for conversation and conflict. This crerates opportunity in each Scene for character-driven Story choices: how does your player choose to respond to the other character?
The player is often not visualized in these types of games, so players can project themselves onto the character, but some stories prefer a specific, visualized player character.

While story paths can intersect, be careful to not allow a player to revisit the same Scene and see the same text, or the game will feel broken. A Player Stat can be used to change the text when returning to a place.

Consider other Player Stats you can track to provide alternative outcomes: objects the player collects, characters the player befriended or pissed-off, that could change later inetractions / access).
Also consider motion effects to enhance the experience, like type-on or shaky text, or fade-away images.

PLANNING THE VISUALS:
Type an ART BACKLOG: Once you have outlined your story and flow diagram, type a list of all the images you will need for your Scenes. Consider hand-drawn / painted elements and / or manipulated photography.

Each Scene needs:

a) At least one BACKGROUND:

  • 1280 x 720, 72ppi
  • CONTRAST: Backgrounds should support charater legibility, with contrasting hues, values, saturation, and levels of detail to the characters in the same sene
  • DEPTH: Look for opportuntiis to represent foreground, middle-ground, and background spaces



b) At least one CHARACTER that is not the player:

  • ALSO 1280 x 720, 72ppi.
  • Include transparent negative space (so the background can be seen behind)
  • Position each character centered on the left or on right side "thirds".
  • 2-5 poses for each character in the Scene. Change poses and faces to support their intended performance




UI ART: button, slider, text, splash art

Plan your project UI art with a consistent look and feel for all Scenes:

  • Images for Function Buttons in the Frame Scenes and Pause Menu (play, quit, etc, try 250x80), short Next Buttons (200x100) and long Choice Buttons (400x90) in the Story Scenes.
  • Background images for the text fields (suggested: namebox 250 x 50 and speachbox 800 x 200).
  • Splash Art (full page images, 1280x720) for the Main Menu, Win and Lose Scenes.

ALL art files for Unity should be PNGs, 72ppi.

If you are using DISCORD for team communication, do not use Discord as the main storage for your images-- a GOOGLE DRIVE for final images is better. BEST is to just drag the PNG files right into Unity! If you DO use Discord, do not download as WebP. Click "Open in Borwser" to get the actual PNG, and download that.

PRACTICE ZERO-DRAFTING!: Please draft all art as super-quick roughs the first week, so the entire game can be implemented and revised WHILE the more polished art is still being produced!

AUDIO: Background music and ambient sounds can add a lot to a Scene's mood and immersion, if you have someone on your team willing to make them! At the very least, choose one event in your game to emphasize with an audio event, like a creaky door opening.


IMPORTANT: Your Unity Project Lives in a Folder on your Computer!

All objects in the Unity Assets folder, like scripts, images, and audio, are just files on your computer in the Project folder!

Want to change an asset directly?

  • RightClick a file in the Unity Project panel > Assets folder to find them outside of Unity using Windows Explorer, or the Finder on a Mac.
  • Then you can edit them directly using any tools you choose.
  • Save the results, and the changes are applied back in Unity.

IMAGE ASSETS can be easily updated:

  • Drop the new file version with the same name into the Assets folder outside of the Unity Editor.
  • The new file replaces the old in all Scenes, and any Unity parameters on the original are automatically applied to the new version!
  • For this reason, you should NOT have multiple copies of an image or sound file in your Assets folder: only the newest version that replaced the previous one.




UNITY CANVAS EXPLAINED:

UNITY CANVAS:

A Canvas is a 2D space to hold UI Game Objects like Text, Images, and Buttons.

Some Key Canvas Features:

  • Create the Canvas by going to GameObject > UI > Canvas, or right-click the Hierarchy to choose the UI options like Canvas, Image, Button, etc.
  • When a Canvas is created, an Event System is also created in the Hierarchy, to allow UI Buttons to function. If your Buttons do not work in a Scene, the first thing to chek is whether the Event System is present in the Hierarchy.
  • In the Scene view the Canvas appears over-large, but in the Game view the Canvas maps over the visible screen and tracks with camera movement (as long as the Render Mode is set to the default "Screen Space - Overlay").



  • For reliable placement of UI elements:
    1. Set the Game view to display at 16:9 Aspect.
    2. Select the Canvas to change Canvas Scaler > UI Scale Mode = "Scale with Sreen Size" and set he Refrence Resolution: X = 1280 and Y = 720.
  • UI Objects can be static (display text or image and not change) or be updated by a script.
  • Buttons get OnClick() Event Triggers to activate ("call") a script on another object (typically script on the GameHandler).



  • For UI Images, do NOT drag PNGs directly into the Hierarchy. Instead, create UI > Image and then drag the PNG into the Inspector slot.
  • The Canvas Hierarchy display order is vertical: elements lower in the Hierarchy under Canvas display in the Scene on top of elements that are higher up in the Canvas.
  • Canvas Text elements can display blurry in the Scene view if the physical size of the Game view window is small. Increase the Game view to the full intended size to see them more clearly. If they still look blurry, increase the text object Rect and Font size to at least 30, and then scale down to intended size in Canvas. NOTE: This is less of an issue with Text Mesh Pro .


 

UNITY VISUAL NOVEL: SUMMARY OF DIALOGUE SYSTEM

originally based on this Source video (poor recording, but good basics

In this Branching Narrative Unity project, each story Scene gets a UI > Canvas object containing Image, Text, and Button elements.



Each Scene gets one long C# script, named for the scene, with all of the Story Units and including:

  • Actions (player hits "Next" or Spacebar to follow a part of the story)
  • Interactions (player chooses between option buttons to direct the story).


This script is applied to the Canvas object, and then its script slots are populated in the Inspector with the Image, Text, and Button elements in the Canvas, to control when they appear on screen.


These scripts use one critical int variable to progress the story: "primeInt".
This integer drives the forward movement of the story by incrementing every time the Next() function is called through the "Next" button.


primeInt is the index of the entire story scene.
Each Story Unit (a character name and what they say) is assigned a primeInt value to find it ("if primeInt == [this number], play this story unit").

  • The Next() function automatically adds 1 to primeInt, so hitting "Next" goes to the next integer: 2, 3, 4, etc. This is good when a designer wants a sequence of "Next" buttons to just passively follow a conversation.
  • BUT, when managing multiple choices, a choice might lead to a primeInt number many integers away by assigning the a value below it. For example, set primeInt = 39 to make the Next button go to 40.

Each story unit is a series of on-off switches for each Canvas element in the scene.
Want to switch which character pose Image is visible?
Just turn one off: ArtChar1.SetActive(false);
and turn the other one on: ArtChar2.SetActive(true);

a scripted-scene can be as long or as short as you want, but should feel substantial, with at least 10 lines of dialogue and at least one dialogue choice (that lets the plyer define character) before the next Scene choice (that lets the player chooe the next desination).

Because this game is made in Unity, each unit can include animation, particles or other VFX, ambient audio, or any other enhancement. AND, best of all, you can always include a link that goes to a new Unity Scene with entirely other interaction as an interlude from the story (3D or 2D exploration, combat, a puzzle, etc), before returning back to the Visual Novel format.

        }

                Time.timeScale = 1f;
                GameisPaused = false;
        }

        public void SetLevel (float sliderValue){
                mixer.SetFloat("MusicVolume", Mathf.Log10 (sliderValue) * 20);
                volumeLevel = sliderValue;
        }

        public void RestartGame(){
                Time.timeScale = 1f;
                SceneManager.LoadScene("MainMenu");
                // Please also reset all static variables here, for new games!
        }

        public void QuitGame(){
                #if UNITY_EDITOR
                UnityEditor.EditorApplication.isPlaying = false;
                #else
                Application.Quit();
                #endif

[OPTION A] HOW DO I ADD A SOUND EFFECT (SFX) TO MY GAME?
Do you want a door creak sound every time your player changes rooms?
Or a ghostly cry when the ghost appears?

1. RECORDING:
Sound effects can be recorded with your phone, and made with mouth sounds or "foley" work (banging / rubbing / dropping things in the real world-- stay safe!).

The file type should be WAV for Unity
(the prefered format for short audio clips: Waveform Audio format).
If your phone makes an m4a file, please use an online convertor
(search on Google: M4A to WAV).

2. EDITING:

    a. Edit your sound effect .WAV file in your prefered program (Audacity is free).
           Be sure to remove any extra time before and after the desired sound.

    b. Export the shorter clip as a .WAV.

3. UNITY SCRIPT:
In a Scene you want this clip to play:

  1. Open the Scene#Dialogue script in your script editor.
  2. In the top class variables, find and uncomment this line:

      public AudioSource audioSource1;

  3. Then find a Story Unit when you want the clip to play,
    and add this bold line command to the content of that Story Unit:

      else if (primeInt == #){
        audioSource1.Play();
      }

  4. Save your script, return to Unity to compile. If there are no errors, continue!

14. Add an Audio file to your Scene:

  1. Create an Empty G.O., name it "AudioManager"
  2. Drag an audio file onto this object to add it as an Audio Source Component
    .
  3. Under the AudioSource Component > Output hit the hotspot to add the MyMusicMixer.
  4. the AudioManager object into the Project panel to make it a Prefab.
    Now you can easily add the Prefab to other Scenes, where the music file can also be changed.

     


4. UNITY SCENE:

  1. In Unity, import the final WAV clip into the Project panel > Assets folder (for example, a door-knock sound effect).
  2. In the Scene you want this clip to play, Right-click the Hierarchy to create a new Empty Game Object.
  3. In the Inspector, name it "AudioManager" and Reset Transforms.
  4. Right-click AudioManager to create a child Empty Game Object.
  5. In the Inspector, name it for the desired effect, (example: "SFX_Knocking").
  6. Drag the .WAV audio clip from the Assets folder onto the child game object in the Hierarchy. In the Inspector, you will see the audio clip has been added as an AudioSource Component.

  7. In the AudioSource Component settings:
    Set "Play on Awake" = off, so the sound does not play at Scene start.
    Set Output to your MyMusicMixer > Master (if you completed the PauseMenu).

  8. Select the Canvas object in the Hierarchy to see the Story#Dialogue script you modified in the Inspector.
  9. Scroll down to find the new AudioSource1 empty script slot at the bottom of the Inspector.
  10. Drag the child Game Object from the Hierarchy into this empty script slot (because it is named for the audio clip, the name will appear in the script slot).

Hit [Play], and when you reach that Story Unit, the audio clip should play!
If it does not, confirm that Unity audio is turned on (in the Game view, top bar)

To use the same sound effect in multiple Scenes:

  • Drag the AudioManager into the Project panel > Assets > Prefabs folder to make it a Prefab.
  • Drag the Prefab from the Prefabs folder into the Hierarchy of those other Scenes.
  • Repeat step 3 in those Scenes: edit the script to use an AudioSource object.
  • Drag the child Game Object from the Hierachy into the empty Script slot.

Learn how to Trigger Multiple Audio Files (essentially, just make more child game objects, each with their own SFX audio clip and named for that clip, to drag into more script slots)

Music or other ambient sounds, as .MP3 or .WAV, can greatly enhance the mood of scene.

If you don't have anyone on the team with music composition experience:
You can make decent ambient music yourself using an online piano, playing only black keys (major scale).
Try this virtual piano (register for a free account to be able to download your creations).

  5. This adds a new AudioSource component-- see it in the Inspector.
  6. In the AudioSource component settings:
    Set Output to your MyMusicMixer > Master (if you completed the PauseMenu).

PART 12. ADDING PLAYER STATS FOR VARIETY:

Player stats allow story designers to track player choices and deliver custom outcomes. In Unity player stats are added as "static" variables to the GameHandler, so their values persist between Scenes, and so they can be changed and tested by other Story scripts.

For example:
We can track player health with an integer:

1. ADD: To GameHandler.cs script: add a stat "playerHealth" of type = integer:

          public static int playerHealth = 10;


2. CHANGE: To Scene#Dialogue.cs scripts: change this stat:
Reduce it when the player is injured or hungry:
          GameHandler.playerHealth -= 1;

Increase it if the player is healed or fed:
          GameHandler.playerHealth += 1;


3. TEST: To GameHandler.cs script: test the playerHealth stat in the Update():
If the stat ever goes below 1, the player "dies" (change Scene to SceneLose).
          //in the GameHandler
          if (playerHealth <= 0){
               SceneManager.LoadScene("SceneLose");
          }

          //in a story unit script (where primeInt 40 warns of low health):
          if (GameHandler.playerHealth <= 2){
               primeInt =39;
          }





ANOTHER EXAMPLE:
The simplest stats to track are BOOLEANS that simply allow or disallow some part of your game.

Imagine the following scenario:
      A game set in a house with two main story tracks: Going to the attic or going to the basement.
      After multiple Scenes, each track offer the player a chance to change their mind and choose the other track.
      Now, imagine the player has first chosen the basement, but before the final choices that lead to victory or death they are given a chance to change their mind and visit the attic, and they take it.
      So they follow the attic track, and at some point are presented the chance to change their mind and go back down to the basement, and repeat that content.
      Being able to switch to the basement makes sense if they had started in the attic, but being able to go there AGAIN feels silly to a player they has already been to the basement. It feels like the game does not know what the player has been doing. We want that basement-option in the Attic track to not be available if the player has already been to the basement.

      To solve this basement case, we make 3 changes to the scripts:
1. ADD a static boolean "canBasement" in GameHandler.cs, set to "true" by default:
          public static bool canBasement = true;


2. CHANGE the stat: in a Basement Scene#Dialogue.cs set the canBasement bool value false (because we are starting in the basement):
         GameHandler.canBasement = false;


3. TEST the stat: in the Attic Scene#Dialogue: add an if-condition around the visibility of the button that switches to basement (ButtonChangeToBasementTrack):
          if (GameHandler.canBasement == true){
                    ButtonChangeToBasementTrack.SetActive(true);
          }
          else {
                    ButtonChangeToBasementTrack.SetActive(false);
          }


Now, when the player starts in the basement, the canBasement bool is set to false.
If the player changes their mind when offered the chance to jump to the Attic track, the Attic track will NOT offer the option of jumping back again to the basement, because the if-condition hides that button.

Naturally, this should also be done in reverse as well:

    Add a canAttic static bool in the GameHandler.
    Change the stat to false in the Attic track.
    Test the stat in the Basement track to decide if a button back to the Attic track is allowed.




Again, every STAT needs THREE TYPES OF SCRIPT CHANGES:

STAGE 1: ADD: GameHandler.cs script: Add a static variable (optionally, functions to change and check it).

STAGE 2: CHANGE: Dialogue.cs scripts: One or more places that change this value in Story Units (primeInts)

STAGE 3 TEST: Dialogue.cs or GameHandler.cs: One or more places that test this value to create consequences, like which Story Units are available, which room-buttons are visible, which endings the player gets, etc.

CRITICAL NOTES:
1. GAMEHANDLER UQBIQUITY: A copy of the GameHandler Prefab must be in EVERY Scene to preserve static variables, including Menus!

2. RESET ON RESTART: Make sure your GameHandler Restart() function, used for the Pause Menu and the End Scenes, includes a RESET of any stats!
Set BOOLEANS back to false, INTs back to their starting value. etc, so progress does not carry over to a restart!!

PART 14. SOME COMMON ERRORS AND SOLUTIONS:

[a] SCRIPT NOT COMPILING?
Check the console for errors and line numbers! Most errors are about:

  • Forgetting to end a line with a semicolon (;)
  • Not completing nesting brackets ({ }), especially at the end of the Next() function.
  • Having the wrong class name: When you duplicate and rename a script for a new Scene, be sure the Class name at the top of the script matches the file name!
  • Having the wrong variable name. A variable must be identical everywhere in the script, including capitalization.
  • Using GameObject vs gameObject incorrectly. "GameObject" = the type, "gameObject" = a specific instance.
  • Using the wrong accessibility. To reference a function in another script, we need to set that function public!
  • Accidentally duplicating a script: there can only be one GameHandler!


[b] OBJECTS APPEARING IN SCENE BUT NOT THE GAME VIEW?
In 2D games we need to manage the display order of elements.

In a Canvas, the higher an image is located the further back it appears.
(For action game Tilemaps and regular Sprite objects set Sprite Renderer > Additional Settings > Order in Layer in the Inspector: the lower the number, the further back it appears.)

Also, be sure to check that your object Z-position is not behind the Camera!

    A 2D game Camera should be Position Z = -10.
    All of your 2D objects should be Position Z= 0.


TRYING TO LOAD A MISSING FUNCTION ON A BUTTON ONCLICK()?
To make a button work:
  1. Select the button and in the Inspector add an OnClick() function.
  2. Drag an object with the relevant script onto the OnClick() None slot (in the lower left, like GameHandler).
  3. Search the Function list (to the right) for the relevant script and choose the function. If you cannot find the function in the list, chances are you have not yet set the function access to public in the script itself.


BUTTONS NOT WORKING-- NOT EVEN ROLLOVER COLOR CHANGE?
Make sure your Scene Hierarchy has the EventSystem game object.
An EventSystem is auto-created when adding buttons to a scene, but can be easy to accidentally delete.

OR, you may have a Canvas Image layer overlapping "in front" of the buttons, blocking their functionality.
Buttons need to be "up front"-- i.e., at the bottom of the revelant Canvas area.
Nothing can overlap in front of them: nothing in a Canvas can be below them in the Hierarchy (except the PauseMenu, which is SUPPOSED to block them when it is enabled).


HOW TO CREATE CANVAS "FOLDERS":
In general, avoid nesting Canvas objects or Empty Game Obects inside a Canvas.
Instead, create a UI > Image object, set Color Alpha value = 0, and turn Raycast Target = off.


DOES HITTING SPACEBAR CAUSE THE DIALOGUE TO SKIP STORY UNITS?
The Spacebar can make the Next() function activate twice, if the button Navigation parameter is set to the default "Automatic".
To fix this:

  • Find the NEXT button Prefab in the Project panel.
  • Doubleclick the prefab to open it in Prefab space.
  • In the Inspector, under the Button component, set Navigation = None.


 
HOW TO REPLACE ART FILES WITH REVISED ART:
A key idea in game development is to get a rough version of the entire game working as fast as possible, to playtest and revise.

For art, this means starting with super-fast, super-rough versions of every asset, and then replacing the art with revised versions as those assets are created.

Starting with fast, rough art saves a lot of time, because a lot of art may need to be changed during the production /playesting process, as the team realizes a Scene needs to be added, cut, or changed completely. It is easier to change art assets that you did not put a lot of time into!

It is EASY to swap in the better art. Just follow these two guidelines!:

1. NAMES ARE CRITICAL:

  • When first importing rough PNG art into your game, name each asset by the final intended name!
  • For example a character PNG might be called: "char_Sam_happy.png".
  • This is the only name you ever want to use for this asset.
  • Also, there should only ever be ONE COPY of each art asset in your Project panel, and it should have this final name.

2. REPLACE IN THE FOLDER, NOT IN UNITY:

  • The revised art PNG needs to have the SAME NAME as the old art, to replace it!
  • Do NOT just drag the new art into the Unity Project panel. Instead of replacing the old art, Unity will make a new copy with the number (1) at the end. This is not what you want!
  • Instead, do this:
    1. Open your Unity project FOLDER (Window on PC, Finder on Mac), and find the folder that contains the art asset you want to replace.
    2. Drag the revised art with the SAME NAME into this folder.
    3. When prompted, confirm you want to REPLACE the old art!


If you do this right, you will not need to drag the revised art into any Scenes to replace the old art: the new art will automatically replace everywhere the old art was used!

Want to use multiple audio clips in a scene?

Create variables to hold AudioSource components and trigger each as needed.
The sample script below uses keyboard input to trigger varied sounds, but we can use Story Units to trigger instead.

1. Create 3 SFX (sound effects) .WAV files or try these 3 Sample Audio Clips.

To make your own SFX:

  1. Use a voice recording app on your phone and use your mouth or objects around you to generate sounds.
  2. Download and use a free website to convert the results from .M4A format to .WAV, for editing (Google M4A to WAV converter).
  3. Edit the clips so each includes only one sound with no space before or after. Audacity is a free program commonly used for quick audio editing.
  4. In the Unity Project panel import your audio clips (the following assumes an example with 3 WAV clips).


2. Create a GameObject > Empty GameObject.
In the Inspector name it "MyAudioManager".

  1. Rightclick MyAudioManager to add another Empty GameObject.
    Name it "AudioClip1"
  2. Add an AudioSource component to AudioClip1:
    Select the child empty GameObject, and in the Inspector add an AudioSource.
    Under AudioSource set "Play on Awake" = off. Make sure Looping is also off.
    (NOTE: dragging an audio clip from the Project panel onto a GameObject will also create a single AudioSource component)
  3. Duplicate AudioClip1 twice, for a total of three child GameObjects with AudioSource components. Name the other two AudioClip2 and AudioClip3.
    (NOTE: Normally, these would all be named for the audio file name)
  4. Drag each audio clip into the Source slot of three AudioSource components.
    For each AudioSource, turn off "Play On Awake" (so they do not auto-play at Start)


3. Create a new C# script named "AudioTest".
  1. Add the content below, Save, and click Unity to compile.
  2. Add the script to the AudioSource Game Object.
  3. Finally, drag each AudioSource component into each script slot (just click and drag from the rolldown name-bar of each into the slots).
Hit [Play] and hit the keyboard numbers [5], [6], and [7] to test!

[G]: EFFECT: CHARACTER BOUNCE:
Make a little bounce for your character art! For example, on all pose changes, or just when a character is super-happy (or frightened?).

PART 1: Create the bounce script and add it to the Story Scene Canvas:

a) RightClick the Project panel > Assets folder to create a new C# script.

  1. Name it "CharacterBounce.cs".
  2. Delete all content and replace with the content below.
  3. Save, and in Unity wait for the code to compile.

CharacterBounce.cs:
using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UI;

public class CharacterBounce: MonoBehaviour{
     public AnimationCurve curveMove = AnimationCurve.EaseInOut(0f, 0f, 1f, 1f);
     float elapsedMove = 0f;
     public GameObject[] charImages;
     public Vector3[] startPos;
     public float bounceAmt = 5;
     bool timeToBounce = false;

     //establish initial positions:
     void Start(){
          for (int i = 0; i < charImages.Length; i++){
               startPos[i] += charImages[i].transform.position;
          }
     }

     //for testing, hit [b]:
     void Update(){
          if (Input.GetKeyDown("b")){
               BounceMe();
          }
     }

     void FixedUpdate () {
          if (timeToBounce==true){
               // Tween Move:
               for (int i = 0; i < charImages.Length; i++){
                    float height = curveMove.Evaluate(elapsedMove);
                    Vector3 newPos = new Vector3(startPos[i].x, startPos[i].y + (height *bounceAmt), startPos[i].z);
                    charImages[i].transform.position = newPos;
               }
               elapsedMove += Time.deltaTime;
               //reset elapsedMove:
               if (elapsedMove >= 1){
                    elapsedMove = 0;
                    timeToBounce = false;
               }
          }
     }

     public void BounceMe(){
          timeToBounce = true;
     }
}

//To use this script in a Story Unit, add: //GetComponent<CharacterBounce>().BounceMe();

//NOTE: To do this bounce with two characters that are on screen at the same time, would need two BounceMe functions, and two sets of arrays


b) Add the script to the Canvas in a Story Scene, below the Story#Dialogue script.
Change these settings in the Inspector:

1. Click on the Animation Curve Rectangle to open the editor.

We want to change this to a bounce curve:

  1. Right-click the center of the line to [add a new key].
  2. Move the new middle key: value=1, time=0.25
  3. Move the right-most key: value=0, time=0.5
    (right-click the control points to Edit precisely).
    This should make a nice, fast, up-and-down bounce.

2. Set the CharImages array to the number of character images you want to bounce (typically, 3-5).

3. Drag the character images from the Hierarchy into the slots that are created.

4. Set the StartPos array number to the same as CharImages.

Hit [Play], display a character that was added to the array, and hit [b] to test!

[I]: EFFECT: MAIN MENU PARALLAX: (source)
For MainMenu art, where forground overlays background (like looking up at trees that overlay the sky), make the forground move against the background when the mouse moves, to create a parallax effect.

a) Set up the MainMenu art:

The MainMenu should be set up in the Canvas as normal, with one big change:

  1. Instead of a single background image, include at least two: a sky (with clouds) and at least one layer of overlaying trees / architecture / etc.
  2. The sky PNG that won't move can be 1280 x 720 as usual, but the foreground image PNG/s that will move must be larger: 2000 x 1000 recommended.
  3. Create the Images in the Scene as UI > Image, and add each PNG to the Inspector SourceImage slot. In the Canvas, the more "in front" an image is, the lower it's position in the Hierarchy list.

b) Create a new script "UIParallax.cs". Add the content below, save, and return to Unity to compile.

c) Apply the new script to the Foreground UI image in the Canvas.
If there are multiple forground layer images, give each a different _parallaxStrength value in the Inspector (lower values for images further back. Try 40 and 80).

UIParallax.cs:
using UnityEngine;
using System.Collections;
using System.Collections.Generic;
using UnityEngine.UI;

public class UIParallax : MonoBehaviour {

     public float parallaxStrength = 80f;
     public Vector2 parallaxClamp = new Vector2(100,100);

     private Vector2 startPos;
     private Camera cam;

     void Start (){
        //load starting position and camera
        startPos = transform.position;
        cam = Camera.main;
     }

    void Update (){
       //Get mouse position on screen
       Vector2 mousePos = cam.ScreenToViewportPoint(Input.mousePosition);

       //LERP to mouse position
       float posX = Mathf.Lerp(transform.position.x, startPos.x + (mousePos.x * parallaxStrength), 5f * Time.deltaTime);
       float posY = Mathf.Lerp(transform.position.y, startPos.y + (mousePos.y * parallaxStrength), 5f * Time.deltaTime);

        //Clamp within bounds
       posX = Mathf.Clamp(posX, startPos.x - parallaxClamp.x, startPos.x + parallaxClamp.x);
       posY = Mathf.Clamp(posY, startPos.y - parallaxClamp.y, startPos.y + parallaxClamp.y);

       //Update current pos to new pos
       transform.position = new Vector3(posX, posY, transform.position.z);
   }
}


The foreground image/s should now move with the mouse.
Hit [Play] to test.
