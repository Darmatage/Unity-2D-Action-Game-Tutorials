UNITY - Local Multi-player
Multiplayer games offer opportunties to drive engagement with competitive or cooperative play. Unity used an input system for many years that still works, but they are shifting over to a new, easier to implement system that is explained below. NOTE: this only works for Unity 2019 and above.

SETTING UP:
Create a new Unity 3D project (these notes and scripts can be adapted for 2D game as well). Import into the Asset folder all three of the scripts below.

 
Local Multiplayer Input System (Video Source)

[1]. Import and install the package:
a. Open Window > Package Manager
b. At top click Advanced > Show Preview Packages
c. Scroll down to Input System (preview 0.9.0), hit [install]. For popup choose [yes]
d. Restart the Unity Editor so inputs are read correctly (even if Unity prompts this, do it again).
NOTE: Changed your mind and want the old Input System back? The switch is under Edit > Project Settings > Player > Other Settings > Active Input Handling. Choose Inoput Manager (Old). This will restart the Editor, then ask if you want to switch back to the new. Say no if you still want the old).

[2]. When Unity restarts, add Devices and Add Inputs Asset to your project:
a. Plugin desired devices (like USB XBox Controllers).
b. Unity Assets menu > Create > Input Actions (bottom) and Name ("PlayerInput")
c. In Inspector, hit [Edit Asset]. The Input Actions editor opens, and can be docked next to scene and game. All inputs will be entered here in these three columns, going from left to right.

[3]. Add Inputs:
a. Left column "Action Maps": hit [+]. Name it "Player". Keep it selected.
b. Middle column "Actions": DoubleClick "New Action" to rename to "Move." (NOTE: in your script this will be a function OnMove). Hit Enter and keep selected.
c. Right column "Properties": set the device: For a joystick, set "Action Type" from "button" to "Value," and "Control Type" to "Stick" (or, if you plan to use both sticks, try "Axis" for the second one).
d. Back in middle column, select <No Binding> so that in the right column we see binding properties. Click Path rolldown and hit [Listen]. On the first XBox controller, wiggle and click the left joystick and choose "LeftStick [Gamepad]". If this does not work, hit the [T] and type it in: <Gamepad>/leftStick
e. Make other actions: mid-col Actions, hit [+] name "Jump" (NOTE: in your script this will be a function OnJump). Rt-col Properties, leave "Action Type" as "Button". Mid-col Actions select <No Bindings>, rt-col Properties click rolldown, hit [Listen], hit desired controller button, and choose the top option.
f. At top of Input Actions Editor, hit [Save Asset] (super important, after every use!)

[4]. Add Player Input component to your Player GameObject:
a. Create a cube and name it Player (or, if you are using a custom mesh, parent that mesh under an empty GO and name that Empty GO "Player").
b. In Inspector hit [Add Component], add Input > Player Input and apply to the Player.
c. Note the empty Actions slot at top. Drag PlayerInput Input Actions asset from Project into that slot.
d. Set Default Action Mode rolldown to our Action Map: Player.
e. (Be sure "Auto-Switch" is unchecked -- not available?).

At the bottom of the component we can see actions this component will allow. Now that we have set the Action Map, click off the Player object and then select it again to see those actions update with Move and Jump. These are the Functions we will need to define in the PlayerController script!


[5] Create the PlayerControl script:
RightClick the Project panel to create a new C# script called PlayerControl.cs. Add to Player.
DoubleClick to edit:
PlayerControl.cs:
using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class PlayerControl : MonoBehaviour {

    Vector2 i_move;
    float moveSpeed =10f;
    float JumpSpeed = 500;
    private Vector3 startPosition;
    private Rigidbody rb;

    void Awake (){
        startPosition = transform.position;
        rb = GetComponent<Rigidbody>();
        }

    void Update() {
        Move();
        }

   private void Move(){
        Vector3 moves = new Vector3(i_move.x, 0, i_move.y) * moveSpeed * Time.deltaTime;
        transform.Translate(moves);
        }

   private void OnMove (InputValue value) {
        i_move = value.Get<Vector2>();
        Debug.Log("Moving");
        }

   private void OnJump(){
        if (transform.position.y <= (startPosition.y + 1)){
            rb.AddForce(Vector3.up * JumpSpeed);
            Debug.Log("Jumping");
        } else {
            Debug.Log("I'm too high to jump." + startPosition + transform.position);
        }
    }
}


[6]. Create the Player Prefab and Player Input Manager:

a. Drag the Player GameObject into the Project to create a Prefab.
b. Delete the original from the Hierarchy.
c. Create a new Empty Game Object, name it PlayerInputManger

d. [Add Component] and add Input > Player Input Manager component.
Note options for "Join Behavior" and "Limit Number of Players" (leave as default for now)
e. Drag the Player Prefab into the waiting slot.

Hook up 1 or more controllers and start playing!

NOTE 1: To use a different prefab for each player, we can either:
1. Turn off the Input Manager and drag copies of the prefab into the Hierarchy. Swap out the meshes for each. As long as the number of prefabs in the scene matches the number of controllers, they will be automatically assigned, one to each!
2. Have all prefabs include multiple meshes, hide all but one, and include code that switches which one is visible (pColorNames.cs, below, can be adepated for this purpose).
3. From video source notes: In PlayerInputManager, change the Join Behavior to "Join Players Manually".
"Call JoinPlayerFromUI or JoinPlayerFromAction explicitly in order to join new players. Alternatively, just create GameObjects with PlayerInput components directly and they will be joined automatically."

NOTE 2: How can we call a method constantly by holding the key (for repeated fire, etc)?
If you edit the binding, under where we set the button, see "Interactions". Click the [+], in dropdown choose "Press". Set "Trigger Behavior" to "Press and Release". Click "Save Asset".

 
Make each Player a unique character:
The Player Input component gives each user a unique identifyer, but we can easily ID each player as it is created with a simple Start() script. Make sure to set the Player Prefab tag to "Player" and add this script to change the color, name, and position (so they no longer spawn in the same place). NOTE: Instead of just changing the color, multiple meshes can be associated with the player Prefab, all but one hidden, and this script could change which one is visible.
pColorName.cs:
using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class pColorName : MonoBehaviour {

    private int count;
    private Renderer pRen;
    private float startPosX;

    void Start () {
        count = GameObject.FindGameObjectsWithTag("Player").Length;
        pRen = this.GetComponent<Renderer>();

        if (count == 1){
                pRen.material.color = new Color(0.5f,0.5f,1f);
                this.name = "Player1";
                startPosX = 1;
                Debug.Log("Players = " + count);
                }
       else if (count == 2){
                pRen.material.color = new Color(1f,0.5f,0.5f);
                this.name = "Player2";
                startPosX = -1;
                Debug.Log("Players = " + count);
                }
       else if (count == 3){
                pRen.material.color = new Color(0.5f,1f,0.5f);
                this.name = "Player3";
                startPosX = 3;
                Debug.Log("Players = " + count);
                }
       else if (count == 4){
                pRen.material.color = new Color(0.7f,0.2f,0.7f);
                this.name = "Player4";
                startPosX = -3;
                Debug.Log("Players = " + count);
                }

        transform.position = transform.position + new Vector3(startPosX,0,0);
        }
}

 
Better Jumping:
It is always nice to add an improved jumping physics:
BetterJump.cs:
using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class BetterJump : MonoBehaviour {

    public float fallMultiplier = 2.5f;
    public float lowJumpMult = 2f;
    Rigidbody rb;

    void Awake () {
        rb = GetComponent<Rigidbody>();
    }

    void Update(){
    if (rb.velocity.y < 0) {
        rb.velocity += Vector3.up * Physics.gravity.y * (fallMultiplier - 1) * Time.deltaTime;
    } else if (rb.velocity.y > 0 && !Input.GetButton ("Jump")){
        rb.velocity += Vector3.up * Physics.gravity.y * (lowJumpMult - 1) * Time.deltaTime;
        }
    }
}
