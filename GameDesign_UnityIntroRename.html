UNITY - Intro to the Main Panels
This page covers an introduction to the key Unity panel names and locations.
 

Welcome to the Unity interface!

This is a tour of the 6 key interface panels:
(1) The Project, (2) the Console, (3) the Inspector,
(4) the Hierarchy, (5) the Scene view, and (6) the Game view.

Note that all of these panels are identified with a name tab.
Look around and find each of them.

Only the Top Menus do not have a name tab, but you may recognize some of them from other programs:
File menu (to save a Scene or the entire Project), Edit menu, Help menu, etc.


The above layout is the default layout for Unity.
All of these panels can be re-arranged by clicking on the name-tab and dragging it to a new location.

If you lose a panel, or want the original arrangement, click on the "Layout" rolldown in the UpperRight corner of the interface.
Then choose one of the options (the original layout is called "default").


 
(1) The Project panel is found along the bottom.
This is where we store all asset files for all levels in the game, including Art (PNGs or FBX), Audio (WAV or MP3), and Scripts (C#).

Note the Project panel is divided into two parts:
The left-hand Stack shows the folder structure of the Project
The larger Contents area on the right shows any files stored in the folder that is selected in the Stack



There are two ways to add content to the Project > Assets folder:
RightClick the empty space on the right-hand side
Drag assets into that empty space.

To create folders to organize your assets:
1. RightClick the Project > Assets empty Content area
2. Choose "Create" at the top to open the Create options
3. Choose "Folder".


Most Unity projects are best orgnized with 4 main folders:
Scripts (for all C# scripts)
Scenes (for all game levels and start and end menu screens)
Media (for Art and Audio folders)
Prefabs (for special objects that can be instanced throughout the project, like the Player).



Note the handy slider in the Bottom-Right corner of the Project Contents area.

Slide this all the way to the left to see contents listed as text, and slide all the way to the right to see contents diplayed as large icons.



Files do not have to go into these folders to work, but it helps to keep the project organized, especially when it is a team project.

META FILES:
If a Project asset file is already used in a Scene and you move the file to a different Project folder, Unity still knows where it is because of the "meta" file that is automatically created for every asset. This also works if you rename a file: the meta file keeps track!
.META files are invisible in Unity by default, but can be seen if you open the Assets folder directly on your computer.


 
(2) The Console panel shares the same space as the Project panel: click the tab directly to the right to view it.
The Console shows error messages with scripts so they can be fixed.
It is also wher we can send "Debug.Log()" messages to help us understand how our code is functioning.




Here I am creating a script by RightClicking in the project > Assets > Scripts folder and choosing Create > C# script.
I then entered a name for the script and opened it in my scripting program, and made an error so we could see what that looks like in the Console:




Errors as specfic as this are AWESOME because they tell us what is wrong, so they are easy to fix!

In this case, on line #12 of "FirstScript.cs" I am missing a semicolon.
All commands must end with a semicolon, and it is common mistake to leave on out.
If I make this fix in the scripting program, Save, and return to Unity, I can expect this error to go away.

 
(3) The Inspector is the column on the right.
This is where we set parameters for any selected Project panel asset/s or Hierarchy object/s.

To understand how it works, let's import an asset:
1. RightClick the empty space of the Project > Assets > Media > Art folder to choose "Import New Asset".
2. Browse your harddrive to find and select the file (in this case, a PNG image of a sassy tree).



Once the asset is in the Project panel, select it to see its default paramters in the Inspector on the right.

In this case, as a piece of Pixel art, the paramters should be set to the following:
           Pixels Per Unit = (from 100 to) 16
           Filter Mode = (from Bilinear to) Point (no filter)
           Compression = (from Normal Quality to) None

           Finally, click [Apply] (towards the bottom).



NOTE: Different Inspector settings are available for the Project panel's source asset and the Hierarchy's object copy.

 
(4) The Hierarchy is the column on the left.
It is a list of all the contents of the open Scene.

By default, the only content of a 2D Game Scene is the default Main Camera.

Let's create an Empty Game Object. We can do this one of three ways:
Click the GameObject menu at the top to choose "Create Empty"
RightClick in the Hierarchy to choose "Create Empty"
Hit [Ctrl/Cmd]+[Shift]+[n].



Select the new Empty Game Object.
Look in the Inspector, and see it is just a Transform: a node to hold Posiiton, Rotation, and Scale information.
Set the Position values to 0, 0, 0 (center of the game world space)

To make this Game Object into something more, we could add Components to it: Physics or any custom scripts to manage desired behaviors.

A Camera in Unity is actually just a Game Object with Camera Components applied to it!
Everything in Unity is just Game Objects with Components!



To start making this Empty Game Object more interesting, we can make it into the parent object of the tree art:
1. Select GameObject and in the top of the Inspector rename it "Tree"
2. Drag the tree art asset from the Project panel into the Hierarchy.
3. In the Inspector rename it "tree_art" and set Position values to 0,0,0 (you can also hit the 3 little dots to the right and choose "Reset Transforms").
4. In the Hierarchy, drag tree_art onto Tree to make it a child of Tree (see image below).

All art in Unity should be put into a Scene this way: as an empty Game Object parent to hold script components (Physics, etc) and a child Art object to hold animation.

 
(5) The Scene view is where we adjust the Position and other Transforms of our Game Objects.
While Objects should only ever be selected in the Hierarchy, they can be Transformed (Moved, Rotated, and Scaled) in the Scene view.

The buttons for Move, Rotate and Scale are in the upper left corner of the interface, and can also be accessed with these keyboard shortcuts:
Move [w] , Rotate [e] , Scale [r]

Also in the upper left corner is a hand icon for panning in the scene.
The shortcut for panning a Unity scene is to click-and-hold the middle mouse button.
To zoom in and out of your Scene, use your middle mouse button as a scrollwheel.



In a 2D game, absolutely every Object except for the Camera MUST be set to Position Z = 0.
This is true for all parents and all children.
The Camera should be set to Position Z = -10, always.
If collisions are not working in game, check that all non-camera objects are set to Position Z = 0.

A quick way to find out if any objects are NOT set to posZ=0 is to temporarily turn off the "2D" button at the top of the Scene view, and Orbit around:
[Alt/Option]+MiddleMouse Click and Drag. In 3D mode there is also a world-orbit gizmo in the UpperRight corner of the Scene, to choose view sides (click a color cone).

Also useful in 3D is the Display Mode rolldown (directly below the name tab).
The default "Shaded" is what we typcally want, but in some circumstances it is helpful to switch to "wireframe" mode.


 
(6) The Game view is where we can see the actual apearance of our assets in the game.
Find the "Free Aspect" rolldown, click it to view options, and choose 16:9 Aspect (good for a standard PC 1280x720 game)

The Game view is entered automatically if we hit [Play] (top center of the interface).

In Play mode we can still switch back and forth between Scene and Game views, and still select assets in the Hierarchy to change their properties in the Inspector, BUT, any changes in a Scene made during Play mode are reversed when we turn off Play (by design).



THAT'S IT!

You now know enough about the interface to get started.
