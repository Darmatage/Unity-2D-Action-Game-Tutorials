TEXT ADVENTURE EXAMPLE #2: Breaking our Interactions into Functions
While straightforward, the long column of nested if-s is hard to edit, and becomes increasingly difficult to manage for a longer game.

A solution can be to break out each interaction into a separate function, with calls to those function in the gameloop at the bottom.

When creating a script with called functions in Python, remember the Order of Execution:
FIRST define the functions,
THEN follow with the code which uses the defined functions:
  • #comments can go on top
  • followed by Import Module statements
  • followed by variables
  • then Define Functions
  • finally, include the code which uses the defined functions

    TEXT ADVENTURE #3a: Location Based Design Using While-Loops
    A helpful way to conceptually organize a Text Adventure game, both for the designers and the players, is to create a grid of regions (lay them out with notecards!) and imagine the game as movement between these regions. These regions can be rooms in a building, different buldings in a town, areas of a forest, etc.

    A simple way to implement a location-based design is to make each location a while-loop, and enter each while looop by changing a variable that designates location.

    TEXT ADVENTURE #3b: Using Python Dictionaries to store player states
    In the following example we use Dictionaries to store information about player progress, including the places in the game where they have already visited.

    Consider this example:
    solved_places = {'a1' : False, 'a2' : False,
                                  'a3' : False, 'a4' : False
                                  'b1' : False, 'b2' : False
                                  'b3' : False, 'b4' : False}

    The curly-brackets { } indicate a Dictionary, with values that can be changed (parentheses would mean Tuples, with values that are immutable). These colon pairs, separated by commas, create a Dictionary of Key-Value pairs including a string key and a value (in this case, a bool that when solved can be used to trigger events in that area or others, etc.). SO, we can ask for solved_places.a1, and print or assign to the paired value.

    To assign a value, or create a new entry:
    dictionaryname ['keyname'] = 'string' or Bool
    EXAMPLE: solved_places ['ai'] = True

    Since pairs are referenced by their key, the order they are added does not matter.

    We also create a Class (a list of attributes which can be used to create multiple game objects of that type) to initialize a player "myPlayer". How can we use this to create NPCs?
