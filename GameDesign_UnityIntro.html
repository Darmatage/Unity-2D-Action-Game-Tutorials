UNITY - Intro to Functionality (Super Artist Friendly)
This page covers an introduction to soem key Unity functionality to get comfortable.
 

Welcome to Unity!

Before following this tutorial, please review the Unity Panels
This tutorial is intended to ease non-programmers into
Unity functionality, and has three parts:
(1) Adding Art to a Game,
(2) Adding Physics interactions,
(3) Adding Code Interactions.


 

PART 0: Get free Unity ID, Install Unity HUB, install the Editor through the Archive: 2022.2.21
Download and install a scripting tool: an IDE (VS Code recommended) and in a new Unity 2D Project add the IDE as the default External Tool:

On a PC: Edit > Preferences > External Tools. Next to External Script Editor click the rolldown to choose "Browse". In the window that opens choose Program Files and then search for your IDE to click and add it (For VS Code: C:\Users\{Username}\AppData\Local\Programs\Microsoft VS Code ).

On a MAC: Unity > Settings > External Tools. Next to External Script Editor click the rolldown to choose "Browse". In the window that opens choose Applications and then search for your IDE to click and add it.





 

PART 1: Adding tree art to the game:
1. Create a new 2D project, note how the Unity project folders exist outside the program, and can be used to manage /replace placeholder art files in the Assets folder.

2. Drag a 64 x 64 pixel tree PNG into the Unity Project > Assets folder. See it appear in the computer folder structure!
3. Drag the tree art into the Hierarchy, to add it to the scene. See how small and blurry it is! (relative to the camera rectangle)
4. Also note the different Inspector parameters for the tree art when it is selected in the Hierarchy (the copy in the game level) vs selected in the Project panel (the source file).
In the Project panel select the source art tree, and in the Inspector set the Pixels Per Unit from 100 to 16, and hit [Apply] to see the Scene view tree grow to a useful size (relative to the camera rectangle).
5. Set Filter Mode from Bilinear to Point (no filter), Compression = None and hit [Apply] to remove the anti-aliasing / blurriness. Now the pixels look sharp!
6. Note the difference between the Scene and Game view tabs. The Scene view is interactive, the Game view just shows the current state of what the camera sees.
In the scene view, hold [space bar] to pan. If you get lost, select the Camera in the Hierarchy, hover the mouse over the Scene view, and ht [f] to center / focus.


 

PART 2: Examine tree art Components, add Physics2D and a Ground

7. With the Hierarchy tree art selected, note the Components in the Inspector: the Transforms and the Sprite Renderer.
See how to Move/Rotate/Scale in the Scene and in the Inspector.
See how to Reset Transforms in the Inspector.
See how to change the Color multiplier in the sprite renderer, and then set back to white.

8. ADD PHYSICS COMPONENTS: We want the tree to be able to fall (Gravity) and to stop when it hits things (Collider). Both of these are physics attributes.
There are two physics engines. The default Physics folder is NOT what we want today-- it is for 3D!
We want only Physics2D for all physics components in a 2D game.
Hit [Add Component] to the bottom of the Inspector to add Physics2D > Box Collider2D, and then again for Physics2D > Rigidbody2D.
Hit [Play] and watch the tree fall (due to the rigidbody, which adds gravity).
Turn off play.
NOTE: any changes made during play mode are lost when turning off play!
This is by design: we don't want destroyed characters to be lost forever, and the side benefit is we can test parameters without fear of messing up the game.

9. Create a ground:
Add a GameObject > 2D Objects > Sprites > Square. Keep it selected.
Use the move tool to drag it to the bottom of the camera rectangle: hit [w] or press the move icon in the Scene view, select the green arrow, drag it down.
Use the Scale tool to make a wide ground: hit [r], select the red axis line, drag to the right to stretch the square on that axis.
With the square still selected, in the Inspector at the top change the name to "ground". At the bottom of the Inspector, hit [Add Component] and add Physics 2D > Box Collider2D.
Also, change the color of the square from white to a darker color: In the Inspector, find Sprite Renderer, and change the color.

10. Hit [Play] to see the tree stop at the box! Turn off Play.
To make the tree stop halfway into the box, to create the illusion of 3D:
Select the Hierarchy tree. In the Inspector find the Sprite Renderer component and change the Order in Layer = 80.
This is how we make objects appear "in front" -- the higher the order in layer, the more in front, Player characters are usually 100, enemies around 80, background layers around 0-40, etc.
NOTE: We do NOT change the position Z value-- all objects in a 2D scene must be position Z=0 (except for the Camera, which stays at z=-10 so it can see all the scene content)
Select the box, and in the Inspector, in the BoxCollider2D component, click the Edit Collider button to see the dot-controls. Drag the top dot-control down to make the collider half as tall.
Hit [Play] again to see the tree stop half-way into the box!

11. MAKE THE TREE BOUNCY!:
RightClick in the empty area of the Project panel to create a 2D > Physic Material 2D. Rightclick the new Physic Material asset to rename it "pm_BouncyTree".
Select pm_BouncyTree and in the Inspector set bounciness = 1.
Drag pm_BouncyTree from the project panel onto the tree art asset in the Inspector. It should now appear in the tree BoxCollider2D component as the Material.
Hit [Play] to watch the bounciness!
